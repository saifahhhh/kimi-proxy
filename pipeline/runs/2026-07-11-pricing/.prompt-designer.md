## Goal

สเปคหน้า **Pricing** ไฟล์ HTML เดียว (inline CSS/JS, ไม่มี backend) แสดง 3 แผน — Starter / Pro / Enterprise — ภายใต้ design system Forward Insight เดิม เพื่อส่งต่อให้ Designer และ Dev ใน pipeline ทำงานต่อได้ทันทีโดยไม่ต้องถามกลับ

> หมายเหตุ: [PLAN]/design tokens ที่แนบมาเป็นของหน้า login run ก่อนหน้า — จะ**ยืมเฉพาะ tokens ระดับระบบ** (palette, typography, spacing, radius/shadow, focus ring) มาใช้ ส่วน layout/states ของ login ไม่เกี่ยวกับหน้านี้

## Plan (phased)

### Phase 1 — Assumptions & Scope (ตัดสินใจแทนมนุษย์)
- ภาษา UI: ไทย (ตาม design system เดิม, Thai UI / English code) · ชื่อ product: **Forward Insight** · สกุลเงิน: บาท (฿) เรียกเก็บรายเดือน พร้อม toggle รายปี (ลด 20%) เป็น mock ฝั่ง client
- ราคา default: Starter ฿0 · Pro ฿990/เดือน (แผนแนะนำ, ไฮไลต์) · Enterprise "ติดต่อฝ่ายขาย"
- ทำ: หน้าเดียว responsive ≥320px, 3 pricing cards, feature list ต่อแผน, ปุ่ม CTA ต่อแผน, toggle รายเดือน/รายปี (JS สลับตัวเลขราคา)
- ไม่ทำ: ระบบชำระเงิน, ฟอร์มติดต่อจริง, i18n, backend ใดๆ — CTA เป็น mock (คลิกแล้วแสดงข้อความยืนยันใน card เดียวกัน ไม่ navigate)

### Phase 2 — UI Elements & Layout
- พื้นหลัง gradient `#0F172A → #1E3A5F → #0E7490` (135°) เต็ม viewport, เนื้อหากึ่งกลาง
- ส่วนหัว: heading "แผนราคา" 24px/700 สีขาว + คำโปรย muted-on-dark, ถัดลงมาเป็น **toggle รายเดือน/รายปี** (ปุ่มคู่ radius 8px, active = พื้น primary `#0E7490`)
- แถว 3 cards (พื้นขาว, radius 16px, shadow ตาม tokens, padding 32px, กว้างการ์ดละสูงสุด ~340px): ชื่อแผน (20px/700) → ราคา (32px/700 + หน่วย 14px muted) → เส้นคั่น → รายการฟีเจอร์ 4–6 ข้อ (16px, ไอคอนถูก SVG สี success `#15803D`) → ปุ่ม CTA เต็มความกว้าง สูง 48px
- แผน Pro: ขอบ 2px primary + badge "แนะนำ" มุมบน, ปุ่ม CTA พื้น primary ตัวขาว · Starter/Enterprise: ปุ่ม secondary (ขอบ primary พื้นขาว ตัว primary)
- ป้าย CTA: Starter "เริ่มใช้ฟรี" · Pro "เริ่มใช้งาน Pro" · Enterprise "ติดต่อฝ่ายขาย"
- responsive: ≥900px เรียง 3 คอลัมน์ (flex/grid) · ต่ำกว่านั้น stack คอลัมน์เดียว, margin ข้าง 16px, จอ ≤360px ลด padding card เหลือ 20px

### Phase 3 — States & Behavior (mock ฝั่ง client)
- default: โหมดรายเดือน active, ราคาแสดงตามรายเดือน
- toggle รายปี: JS คำนวณราคา ×12×0.8 แสดงเป็น "/ปี" (Enterprise ไม่เปลี่ยน — ไม่มีตัวเลข)
- คลิก CTA (Starter/Pro): ปุ่ม disabled + spinner 18px ~800ms → แทนข้อความในปุ่มเป็น "เลือกแผนแล้ว ✓" พื้น success-bg ตัว success · Enterprise: แสดงบรรทัด "ส่งคำขอแล้ว ทีมขายจะติดต่อกลับ" ใต้ปุ่ม
- focus-visible ทุก interactive element ใช้ ring `0 0 0 3px rgba(14,116,144,0.25)` · hover ปุ่ม primary → `#155E75`

## Constraints

- ไฟล์เดียว: HTML + inline `<style>`/`<script>` เท่านั้น ไม่มี dependency ภายนอกยกเว้น Google Fonts (Noto Sans Thai 400/500/700)
- ยึด tokens จาก Design System Forward Insight ทุกค่า (palette/typography/spacing/radius/shadow/focus ring) — ห้ามสร้างสีหรือ scale ใหม่
- line-height ≥1.5 ทั้งหน้า (ตัวไทยมีสระบน-ล่าง) · contrast ผ่าน AA · Thai UI, English identifiers
- AC pattern เป็น ERE บรรทัดละตัวเดียว ไม่มี `" => "` ซ้อนใน pattern, เช็คจากตัวอักษรในไฟล์เท่านั้น (behavior รันไทม์ปล่อยให้ headless browser ของ pipeline เช็ค)
- ห้ามขยาย scope เกิน 3 แผน / ไม่มีหน้าอื่น / ไม่มี navigation จริง
